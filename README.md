## How I work

```text
Problem → Constraints → Plan → Build → Verify → Reflect → Reuse
```

- **产品判断**：先定义用户、约束、成功信号，以及这轮明确不做什么。
- **系统交付**：把前端、后端、数据、AI、异步任务、部署与反馈连接成可运行闭环。
- **AI 编排**：让 agents 承担探索、实现、评审和机械工作，但保留人工决策点、回滚路径与最终责任。
- **验证纪律**：检查真实症状、测试结果、代码差异与公开页面；没有证据时就降低结论强度。
- **知识复利**：把问题、复盘和反馈沉淀为可复用的规则、教程与下一步行动。

## Evidence before adjectives

Profile 中的状态只表达可核验范围：

- `Shipped`：已有公开可访问的产品表面；不代表披露用户规模、商业效果或内部实现。
- `Maintained`：公开内容与系统仍在维护；不代表内容已经同行评审或覆盖完整领域。
- `Prototype`：证明了限定范围内的交互或工作流；不等于生产就绪。
- `Private source`：只提供脱敏案例与公开证据，不暗示源码已公开审计。

涉及 AI 辅助的公开展示、教程生产或工程流程时，归属范围以对应案例的 Role / Evidence / Limits 为准；本页不会把 AI 生成或共同完成的结果改写成个人独立成果。
